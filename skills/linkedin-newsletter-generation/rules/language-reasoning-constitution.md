Language & Reasoning Constitution v1.0 (Complete Edition)

This document governs how reasoning is expressed, not what conclusions are reached. Violations are structural, not stylistic.

I. Semantic Load Doctrine

Rule I.1 — Every Sentence Must Do Work
A sentence is valid only if it performs at least one of:
- Introduces a constraint
- Explains a mechanism
- Narrows scope or applicability
- Exposes or resolves a trade-off
- Eliminates an alternative

If a sentence merely signals intelligence, smooths transitions, or adds emphasis, it is invalid.

Rule I.2 — Compression Over Expansion
When two sentences can be merged without loss of precision, they must be. Writing should feel compressed, not polished.

II. Concept Validity Rules

Rule II.1 — Concepts Must Be Operationalizable
A concept may appear only if you can answer: What does this constrain? What does this change in a decision?

Rule II.2 — No Floating Abstractions
Abstract concepts must be anchored by a constraint or mechanism. Abstractions are compression devices, never endpoints.

III. Authority & Voice Rules

Rule III.1 — Authority Is Structural, Not Biographical
Credibility must emerge from constraint logic, trade-off exposure, and failure explanation—not from experience, titles, or outcomes.

Rule III.2 — First-Person as Instrument, Not Signal
"I" may only introduce modeling choices, framing decisions, or scope limits. If removing "I" weakens authority, the sentence is invalid.

IV. Causality & Explanation Rules

Rule IV.1 — No Explanation Without Friction
Any explanation must surface at least one friction: Capital, time, incentives, information, cognition, regulation, irreversibility.

Rule IV.2 — Failure Before Prescription
If the text implies what should change, it must first explain why current behavior persists and which incentives/constraints sustain it.

V. Trade-Off Discipline

Rule V.1 — Every Preference Has a Cost
Directional statements are valid only if the cost is named and the cost-bearer is identified.

Rule V.2 — Asymmetry Must Be Explicit
If a decision benefits one party more than another, asymmetry must be named.

VI. Scope & Generalization Control

Rule VI.1 — Universal Statements Are Presumed False
Any broadly applicable statement must be localized by stage, context, actor, or constraint set.

Rule VI.2 — Insight ≠ Transferability
If an insight survives only by abstraction, it is likely empty.

VII. Structural Integrity Rules

Rule VII.1 — Paragraph Completeness
Each paragraph must answer: What is happening? Why does it happen? Why does it matter for decisions?

Rule VII.2 — Endings Are Logical, Not Interactive
A piece ends when analytical tension is resolved. No CTAs, questions, or summaries.

VIII. Meta-Rules

Rule VIII.1 — Silence Is a Valid Output
If an idea cannot be expressed without dilution, it should not be written.

Rule VIII.2 — Density Over Volume
Short pieces are preferred because weak ideas collapse under compression.

Enforcement Mechanisms

Deletion Test: Delete a sentence. If nothing breaks, it was ornamental.
Constraint Substitution Test: Replace key noun with concrete constraint. If meaning improves, original was vague.
Counterfactual Test: Ask "Under what conditions would this fail?"
Anonymity Test: Remove author's name. If authority collapses, text fails.

---

Sentence-Level Decision Tree

1. Does the sentence change understanding of a decision? If no → delete.
2. Is it descriptive or explanatory? Description must be followed by explanation.
3. Is causality explicit or inferable? If no → rewrite.
4. Is a constraint named or implied? If no → invalid.
5. Is scope bounded? If no → localize or delete.
6. Does it introduce or resolve tension? If no → delete.

---

Module A — Trade-Off Taxonomy

1. Capital vs Optionality — locking resources reduces future flexibility
2. Speed vs Reversibility — faster decisions increase regret risk
3. Efficiency vs Resilience — optimized systems are brittle
4. Scale vs Control — growth increases coordination costs
5. Precision vs Robustness — fine-tuned models fail under noise
6. Local Optimization vs System Health — subsystem gains harm the whole
7. Information vs Action — more data delays commitment
8. Incentive Alignment vs Autonomy — control improves outcomes but reduces initiative

---

Module B — Thinking Under Uncertainty

U.1 — Irreversibility First: Prioritize by reversibility, not importance.
U.2 — Option Value Preservation: Prefer actions that preserve future choices.
U.3 — Asymmetric Risk Accounting: Name where downside is capped/unbounded.
U.4 — Decision Quality Over Outcome Quality: Evaluate ex-ante, not ex-post.
U.5 — Fragility Identification: Ask what breaks first under stress.
U.6 — Time as a Constraint: Treat timing as a decision variable.

Global Meta-Constraint
If a smart reader disagrees, they should disagree on assumptions—not clarity.
