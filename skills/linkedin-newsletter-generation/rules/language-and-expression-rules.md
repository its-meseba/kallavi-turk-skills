Language & Expression Rules (Extended, rule-based, non-lexical)

Rule 1: Nouns Must Carry Analytical Load
Every noun should represent a constraint, a mechanism, or a decision object.
If a noun doesn't help reader understand what limits action, what causes behavior, or what choice is evaluated → remove/replace.
Abstract nouns acceptable only if they compress known analytical concept ("incentives", "trade-offs").
Test: If noun deleted, does reasoning weaken? If no → ornamental.

Rule 2: Adjectives Permitted Only as Boundary-Setters
Adjectives should narrow meaning, not amplify.
Valid only if: defines boundary condition ("irreversible", "asymmetric") OR differentiates analytically distinct cases.
Adjectives that merely intensify ("strong", "significant") are invalid unless followed by constraint/comparison.

Rule 3: Verbs Must Imply Causality or Choice
Verbs should indicate mechanisms or decisions, not states of being.
Prefer verbs that answer: What causes this? What changes as result? What is being allocated/constrained/foregone?
Static verbs ("is", "seems") acceptable only when introducing condition that will be immediately explained.

Rule 4: First-Person Is a Lens, Not Evidence
"I" introduces how problem is framed, never why reader should trust conclusion.
Allowed only when: defining analytical lens, declaring modeling choice, narrowing scope.
NEVER: establish credibility, reference experience as proof, express emotion/motivation.

Rule 5: Claims Require Constraint or Counterfactual
No claim stands alone. Must be immediately supported by:
- A constraint ("because capital is fixed...") OR
- A counterfactual ("this fails when...")

Rule 6: Avoid Universality by Localizing Validity
When statement could be read as broadly applicable, must be localized by:
- Stage (early vs mature)
- Context (capital-constrained vs capital-rich)
- Actor (founder, investor, regulator)
- Time horizon (short vs long term)

Rule 7: Trade-Offs Must Be Explicit Before Preferences
Express directional view only after:
- Naming what is being sacrificed
- Acknowledging who bears the cost

Rule 8: Explanations Precede Recommendations
If writing suggests what should change, must first explain:
- Why current approach persists
- What incentive/constraint sustains it

Rule 9: Metaphors Are Diagnostic Tools
Metaphors allowed only when they clarify structure.
Acceptable only if: maps directly onto economic/strategic structure, could be removed without changing conclusion.

Rule 10: Paragraphs Represent Completed Thoughts
Each paragraph must contain:
- One observation or claim
- One mechanism or constraint
- One implication or narrowing

Rule 11: Endings Must Resolve Tension, Not Invite Engagement
End when: trade-off fully articulated, implication clear, no further compression possible.
NEVER: questions for reader, invitations to comment, summaries.

Rule 12: Precision Over Completeness
Better to be narrowly correct than broadly vague.
If concept cannot be explained without multiple caveats → narrow scope or exclude entirely.
Silence preferable to dilution.
