1. PROJECT AIM
================================================================================


This is a multi-agent investment advisory system designed to:


- Gather real-time market data across THREE asset classes:
  * Gold & Silver (Precious Metals)
  * Global Stocks & Funds (Shariah-compliant focus)
  * Turkish Stocks & Funds (BIST Katılım focus)


- Process information through a structured pipeline:
  RESEARCH (9 agents) -> DISCUSSION (Dynamic N agents) -> DECISION (1 agent) -> EMAIL
  * Research data combined with YFinance fundamental pool data -> Discussion Agents
  * Discussion outputs -> Decider Agent (incorporating Memory of past actions)


- Deliver regular investment guidance via email that is:
  * Actionable and concise (Goal-based, future-proof, long-term focus)
  * Halal-compliant (Islamic finance constraints)
  * No-Israel verified (ethical investment constraint)
  * Currency-aware (TRY/USD/EUR exposure management)
  * Turkey-specific risk aware


The main system runs once every 3 days and produces a final advisory report sent to the user.

3. PROMPT FILES (CRITICAL)
================================================================================


The .txt files in the prompts/ folder are NOT documentation.
They are AGENT PROMPTS fed directly to the Gemini API.


Each prompt defines:
- The agent's role and responsibilities
- Operating rules and constraints
- Output format/template
- Specific focus areas


RESEARCH AGENTS (9 total):
- 1A, 1B, 1C: Gold & Silver focus
- 2A, 2B, 2C: Global Stocks & Funds focus
- 3A, 3B, 3C: Turkish Stocks & Funds focus
- Each has sub-focus: News (A), Fundamentals (B), Sentiment (C)


DISCUSSION AGENTS (Dynamic Count):
- Operates as a Mixture of Experts
- Each assigned a randomized continuous "agression level" (0 to 1) and JSON-style dynamic investing profile (e.g., Risk Appetite, Rationality, Strategy Consistency).
- Iterate only once.
- Incorporates research reports and yfinance structural financial data.


DECIDER AGENT (1):
- Synthesizes ONLY Discussion outputs (ignores raw research) + Portfolio + Past memory
- Aligns with user's risk profile (default 2.2-2.3 Moderately Aggressive)
- Self-iterates 3 times for quality
- Produces final actionable summary with long-term, goal-based focus


INFERENCE AGENT (1):
- Interactive Q&A mode (run via inference.py)
- Uses last N final reports + optional web search
- Portfolio-aware: can receive current portfolio context via portfolio_loader
- Single prompt: Inference Agent.txt

4. PORTFOLIO AND INFERENCE MODES

PORTFOLIO (manage_portfolio.py):

INFERENCE (inference.py):
- Loads last N final reports for context (default 3)
- Optional --no-context for web-search-only
- InferenceAgent uses Inference Agent.txt and INFERENCE_MODEL (e.g. gemini-2.5-pro)

6. KEY TECHNICAL DETAILS
================================================================================


API CONFIGURATION:
- Research: gemini-2.0-flash (web search grounding)
- Discussion / Decider: gemini-2.5-pro
- Inference: gemini-2.5-pro (INFERENCE_MODEL)
- Web search: Google Search grounding enabled for research and inference agents
- Rate limiting: 10s delay between agents, retry with backoff


PIPELINE EXECUTION (Cron Schedule):
- Pool update run: python update_pool.py (Once in 2 weeks)
- Price update run: python update_prices.py (Once a day)
- Full system run: python main.py (Once in 3 days)

- Skip email: python main.py --skip-email
- Research only: python main.py --research-only
- Discussion only: python main.py --discussion-only
- Decider only: python main.py --decider-only
- Interactive inference: python inference.py  (optional: -n 5, --no-context)
- Portfolio management: python manage_portfolio.py  (optional: --view, --history)


DATA FLOW:
- Full pipeline: ALL data passed in-memory (fresh)
- Partial runs: Load from disk using TODAY's date filter
- Reports named: REPORT_XX_Type_YYYY-MM-DD.txt
- Portfolio: config.PORTFOLIO_DIR, PORTFOLIO_FILE, PORTFOLIO_CHANGES_LOG, PORTFOLIO_HISTORY_DIR
- Logging: config.LOG_FILE (investment_advisor.log); logs/ folder gitignored


EMAIL DELIVERY:
- Gmail SMTP (smtp.gmail.com:587)
- Requires app password (not regular password)
- Final decision report sent as email body
