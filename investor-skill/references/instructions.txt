PROJECT NAME: Investment Advisor Multi-Agent System
LANGUAGE: Python 3.10+
API: Google Gemini API (google-genai SDK)

================================================================================
1. PROJECT AIM
================================================================================

This is a multi-agent investment advisory system designed to:

- Gather real-time market data across THREE asset classes:
  * Real Estate (Turkey-focused)
  * Gold & Silver (Precious Metals)
  * Stocks & Funds (Global, Shariah-compliant focus)

- Process information through a structured pipeline:
  RESEARCH (9 agents) -> DISCUSSION (3 agents) -> DECISION (1 agent) -> EMAIL

- Deliver daily investment guidance via email that is:
  * Actionable and concise
  * Halal-compliant (Islamic finance constraints)
  * No-Israel verified (ethical investment constraint)
  * Currency-aware (TRY/USD/EUR exposure management)
  * Turkey-specific risk aware

The system runs once daily and produces a final advisory report sent to the user.

3. PROMPT FILES (CRITICAL)
================================================================================

The .txt files in the prompts/ folder are NOT documentation.
They are AGENT PROMPTS fed directly to the Gemini API.

Each prompt defines:
- The agent's role and responsibilities
- Operating rules and constraints
- Output format/template
- Specific focus areas

RESEARCH AGENTS (9 total):
- 1A, 1B, 1C: Real Estate focus
- 2A, 2B, 2C: Gold & Silver focus
- 3A, 3B, 3C: Stocks & Funds focus
- Each has sub-focus: News (A), Fundamentals (B), Sentiment (C)

DISCUSSION AGENTS (3 total):
- Each favors one asset class
- Iteratively refine analysis (3 iterations default)
- Use research reports + previous discussion outputs

DECIDER AGENT (1):
- Synthesizes all 12 inputs (9 research + 3 discussion)
- Self-iterates 3 times for quality
- Produces final actionable summary

================================================================================
5. KEY TECHNICAL DETAILS
================================================================================

API CONFIGURATION:
- Model: gemini-2.0-flash (research, discussion, decider)
- Web search: Google Search grounding enabled for research agents
- Rate limiting: 10s delay between agents, retry with backoff

PIPELINE EXECUTION:
- Full run: python main.py
- Skip email: python main.py --skip-email
- Research only: python main.py --research-only
- Discussion only: python main.py --discussion-only
- Decider only: python main.py --decider-only --iteration 3

DATA FLOW:
- Full pipeline: ALL data passed in-memory (fresh)
- Partial runs: Load from disk using TODAY's date filter
- Reports named: REPORT_XX_Type_YYYY-MM-DD.txt

EMAIL DELIVERY:
- Gmail SMTP (smtp.gmail.com:587)
- Requires app password (not regular password)
- Final decision report sent as email body
